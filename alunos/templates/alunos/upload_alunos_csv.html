Importar Alunos via CSV

Utilize esta funcionalidade para adicionar ou atualizar múltiplos alunos de uma vez através de um arquivo CSV.

Baixar Modelo XLSX
Instruções: O arquivo CSV deve conter as seguintes colunas (sensível a maiúsculas/minúsculas) e no formato especificado:
  • escola_nome (Texto, Obrigatório. A escola deve existir previamente no sistema.)
  • Nome (Texto, Obrigatório. Nome completo do aluno.)
  • Data nasc (00/00/0000) (Data, Formato DD/MM/YYYY, Obrigatório.)
  • Cor (Texto, Obrigatório. Valores: {% for choice in Aluno.COR_RACA_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • Sexo (Texto, Obrigatório. Valores: {% for choice in Aluno.SEXO_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • Estado Civil (Texto, Obrigatório. Valores: {% for choice in Aluno.ESTADO_CIVIL_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • Filiacao (Texto, Obrigatório. Nome completo da mãe.)
  • Naturalidade (Texto, Obrigatório. Cidade de nascimento.)
  • uf (Texto, Obrigatório. UF de naturalidade.)
  • cep (Texto, Obrigatório. Apenas números, ex: 38400000.)
  • endereco (Texto, Obrigatório. Nome da rua/endereço.)
  • num (Texto, Opcional. Número do endereço.)
  • bairro (Texto, Obrigatório.)
  • cidade (Texto, Obrigatório. Cidade do endereço.)
  • estado_endereco (Texto, Obrigatório. Sigla do estado do endereço, ex: MG.)
  • telefone1 (Texto, Obrigatório. Telefone principal com DDD, ex: (34)99999-9999.)
  • email (Texto, Obrigatório. Email único do aluno.)
  • rg (Texto, Opcional.)
  • orgaoemissor (Texto, Opcional.)
  • dataemissao (Data, Formato DD/MM/YYYY, Opcional.)
  • cpf (Texto, Obrigatório. Apenas números, ex: 12345678901.)
  • deficiencia (sim/nao) (Texto, Obrigatório. Valores: 'sim' ou 'nao'.)
  • escolaridade (Texto, Obrigatório. Valores: {% for choice in Aluno.ESCOLARIDADE_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • situacao (Texto, Obrigatório. Mapeia para Situação Profissional. Valores: {% for choice in Aluno.SITUACAO_PROFISSIONAL_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • renda (Número Decimal, Obrigatório. Renda individual, ex: 1500.00.)
  • nummoradores (Número Inteiro, Obrigatório. Número de moradores na residência.)
  • numtrabalham (Número Inteiro, Obrigatório. Quantos trabalham na residência.)
  • rendatotal (Número Decimal, Obrigatório. Renda dos outros moradores, ex: 800.00.)
  • valorresidencia (Número Decimal, Obrigatório. Valor da moradia, ex: 500.00.)
  • tempo_moradia (Texto, Opcional. Valores: {% for choice in Aluno.TEMPO_MORADIA_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • tipo_moradia (Texto, Opcional. Valores: {% for choice in Aluno.TIPO_MORADIA_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})
  • como_soube (Texto, Opcional. Valores: {% for choice in Aluno.COMO_SOUBE_CHOICES %}'{{ choice.1 }}'{% if not forloop.last %}, {% endif %}{% endfor %})

Observações:

  • Alunos são identificados unicamente pelo CPF. Se um CPF já existe, o aluno será atualizado; caso contrário, um novo aluno será criado.
  • Campos de data devem estar no formato DD/MM/YYYY.
  • Campos numéricos devem usar ponto (`.`) como separador decimal.
  • Os valores para campos de `CHOICES` (como Cor, Sexo, etc.) devem corresponder exatamente aos valores por extenso listados acima.
